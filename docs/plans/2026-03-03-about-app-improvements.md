## High Priority

### [Layout] H1 ‚Äî Hero section is not theme-aware
**File:** `components/apps/AboutApp.js:33‚Äì35`
**Problem:** Hero uses hardcoded dark gradient (`linear-gradient(135deg, #0a0a1e 0%, ...)`) regardless of `darkTheme` prop. Light theme switches the outer container to `#f7f8fb` but the hero stays pitch black ‚Äî jarring dark-on-light collision.
**Fix:** Add `heroBg` token to theme object `t`. Light: `linear-gradient(135deg, #e8f0fb 0%, #dce8f5 100%)`. Also make `text-white` on `<h1>` and all inline `rgba(255,255,255,...)` values conditional.

---

### [Typography] H2 ‚Äî Bio footnote fails WCAG contrast
**File:** `components/apps/AboutApp.js:99‚Äì102`
**Problem:** `rgba(255,255,255,0.4)` on `#0a0a1e` ‚âà 2.5:1 contrast ratio. WCAG AA requires 4.5:1 for normal-sized text. ‚úÖ **Partially fixed** ‚Äî now uses `t.footnote` (0.4 in dark, #999 in light), but dark mode value still fails. Raise dark mode to `rgba(255,255,255,0.6)`.
**Fix:** `footnote: "rgba(255,255,255,0.6)"` in dark theme object.

---

### [Content] H3 ‚Äî Bio buries the strongest differentiator
**File:** `components/apps/AboutApp.js:87`
**Problem:** Opens with atmospheric framing ("drawn to systems where the margin for error is genuinely small"). Concrete achievements (BISS encryption, Synkriom ML pipeline) appear in sentences 4‚Äì5. Recruiters spend 10‚Äì20s scanning.
**Fix:** Lead with role + domain + achievement: "Software engineer specializing in low-latency broadcast infrastructure and distributed ML systems ‚Äî most recently leading encryption module design for DTV's BISS implementation and architecting an end-to-end ML pipeline at Synkriom." Move atmospheric line to sentence 2.

---

### [Content] H4 ‚Äî Employment gap footnote reads as a red flag
**File:** `components/apps/AboutApp.js:99‚Äì102`
**Problem:** A disclaimer-style footnote draws attention to a 3-month gap. "Preparing for industry roles" reads as unemployed and searching, not intentional.
**Fix:** Remove the footnote entirely (a 3-month gap is unremarkable in a portfolio About page), OR inline it naturally: "Between graduation and Synkriom, I contributed to open-source systems projects."

## Medium Priority

### [Layout] M1 ‚Äî Skills grid breaks at narrow window widths
**File:** `components/apps/AboutApp.js:106`
**Problem:** `grid-cols-2` always applies. At ~400px window width, two-column cards become <180px wide ‚Äî "Distributed Systems" tag wraps inconsistently.
**Fix:** `grid-cols-[repeat(auto-fit,minmax(220px,1fr))]` for fluid reflow.

---

### [Layout] M2 ‚Äî No section headings for Bio or Skills
**File:** `components/apps/AboutApp.js:78, 106`
**Problem:** Only Certifications has a heading label. Bio and Skills have no scannable section anchors. Recruiter can't skim quickly.
**Fix:** Add `<h2>` labels before bio paragraph and before skills grid, using the same `text-xs font-semibold uppercase tracking-widest` style as the cert heading.

---

### [Typography] M3 ‚Äî Subtitle "Software Engineer" same size as bio body
**File:** `components/apps/AboutApp.js:53`
**Problem:** `text-sm` (14px) for subtitle matches `text-sm` for bio prose ‚Äî no visual hierarchy step between them.
**Fix:** Change subtitle to `text-base` (16px) `font-medium`.

---

### [Content] M4 ‚Äî Location string implies India residency without authorization context
**File:** `components/apps/AboutApp.js:55`
**Problem:** "Madison, WI ‚Üí Elgin, IL ‚Üí Bengaluru, India" chain implies currently in Bengaluru. US recruiters may deprioritize without reading further.
**Fix:** Replace with current location + authorization note: "Currently: Bengaluru, India ¬∑ Open to US remote ¬∑ Authorized to work in US" (or whatever is accurate).

---

### [Visual] M5 ‚Äî Profile photo too small (90√ó90px)
**File:** `components/apps/AboutApp.js:41‚Äì42`
**Problem:** 90px is smaller than a LinkedIn thumbnail (110px) or macOS contact card (128px). Makes the hero feel text-heavy.
**Fix:** Increase to 108‚Äì120px.

---

### [Visual] M6 ‚Äî AWS cert card breaks on light theme ‚úÖ Fixed
**File:** `components/apps/AboutApp.js:133‚Äì142`
**Problem:** Text used `text-white/90` and `text-white/50` ‚Äî invisible on `#f7f8fb` background.
**Status:** Fixed ‚Äî now uses `t.certTitle`, `t.certSubtitle`, `t.certHeading`.

---

### [Accessibility] M7 ‚Äî `user-select: none` inherited on bio text
**File:** `styles/globals.css:41`, `components/apps/AboutApp.js:29`
**Problem:** Global `user-select: none` on `body` prevents recruiters from copying email, skills, or bio text.
**Fix:** Add `style={{ userSelect: 'text' }}` to the root div in AboutApp, or add a `.selectable` CSS class applied to all prose content.

---

### [Accessibility] M8 ‚Äî Contact links missing `aria-label`
**File:** `components/apps/AboutApp.js:58‚Äì72`
**Problem:** Screen readers announce "option key GitHub", "in LinkedIn", "envelope Email". No `aria-label`. Email link also missing `rel="noopener noreferrer"` (low risk but inconsistent).
**Fix:** Add `aria-label="Visit GitHub profile"`, `aria-label="Visit LinkedIn profile"`, `aria-label="Send email"` to each link.

---

### [Accessibility] M9 ‚Äî Invalid heading hierarchy (h1 ‚Üí h3, skips h2)
**File:** `components/apps/AboutApp.js:52, 115, 133`
**Problem:** `<h1>` name ‚Üí no `<h2>` ‚Üí `<h3>` certifications. Skill category labels are `<p>` styled as headings. Non-navigable by screen readers using heading shortcuts.
**Fix:** Add `<h2>` for Skills and Certifications sections. Change category `<p>` labels inside cards to `<h3>`. Wrap blocks in `<section aria-label="...">`.

---

### [UX] M10 ‚Äî No Education section despite UW-Madison being a key credential
**File:** `components/apps/AboutApp.js`
**Problem:** "I graduated from UW-Madison" is buried in prose sentence 3. For a 2‚Äì3 year post-grad candidate, the university and degree are important recruiter signals and should be structured data, not inline text.
**Fix:** Add an Education card (same style as cert card) with: University of Wisconsin-Madison, degree, graduation year. Place between bio and skills.

---

### [Typography] M11 ‚Äî `text-[10px]` cert sub-label below legible threshold
**File:** `components/apps/AboutApp.js:140`
**Problem:** 10px CSS is borderline illegible. Not covered by WCAG large-text exemption.
**Fix:** Change to `text-xs` (12px).

## Low Priority

### [Layout] L1 ‚Äî Redundant `mt-4` inside `pb-8` cert wrapper
**File:** `components/apps/AboutApp.js:131‚Äì132`
**Problem:** Outer `pb-8` + inner `mt-4` creates ~48px dead space before the cert card, inconsistent with `py-6` used in bio section.
**Fix:** Remove `mt-4` from inner div.

---

### [Typography] L2 ‚Äî Location line at 45% opacity
**File:** `components/apps/AboutApp.js:54`
**Problem:** `rgba(255,255,255,0.45)` ‚âà 2.2:1 contrast. Not actionable but still readable content.
**Fix:** Raise to `rgba(255,255,255,0.6)` minimum.

---

### [Content] L3 ‚Äî Emoji prefixes on contact buttons are decorative and OS-inconsistent
**File:** `components/apps/AboutApp.js:61, 70`
**Problem:** `‚å• GitHub` (option key symbol) is OS-specific. Emoji rendering varies. Screen readers announce raw emoji text.
**Fix:** Replace with inline SVG icons or remove prefixes.

---

### [Visual] L4 ‚Äî Red (danger color) assigned to "Competencies" card
**File:** `components/apps/AboutApp.js:11`
**Problem:** `COLORS[3] = "#ff453a"` is the OS-wide danger/error color. Competencies card visually reads as an error state to users familiar with OS color semantics.
**Fix:** Remap Competencies to blue accent or indigo. Or use a single accent color for all cards.

---

### [Visual] L5 ‚Äî Hover state implemented via inline JS event handlers
**File:** `components/apps/AboutApp.js:112‚Äì113`
**Problem:** `onMouseEnter`/`onMouseLeave` writing to `e.currentTarget.style` bypasses CSS transition system, not keyboard-focus triggered, and not overrideable by media queries.
**Fix:** Move hover styles to a CSS class with `transition` defined there. Use CSS custom property `--card-accent: ${COLORS[i]}` referenced in CSS rules.

---

### [UX] L6 ‚Äî No scroll affordance for content below the fold
**File:** `components/apps/AboutApp.js:29`
**Problem:** Certifications section may be fully off-screen at default window height with no visual hint that content exists below.
**Fix:** Bottom gradient mask on scroll container (`::after` pseudo-element fading to `containerBg`), or ensure default About window height shows the top of the cert section.
